Mail DNS Records

MX Record [RFC1035]

MX records are one of first record types defined are were given type code 15. MX records advertise an mail exchange serving for a certain domain name. Currently this is the only email server record in use as all other records are marked as obsolete. MX records specificy a preference (16bit integer, lower values equal higher preference) and a domain name that will handle the incoming mail. There may be multiple MX records defined. 

example.com.   IN  1 mail.example.com.

Experimental Mailbox Based Email Routing [RFC1035]

A competing standard called mailbox records was created as an experimental standard. This standard was never put into use. This standard defined record types MB (TYPE7), MG(TYPE 8), MR(TYPE 9). As well as a query type MAILB (TYPE253). This system used the entire mail address when looking up the destination. The address was converted into a domain name by placing the address part as a single label and then the domain. For instance admin@example.com would be converted into admin.example.com. If the email contained special characters these would be escaped using backslashes. So firstname.lastname@example.com would be firstname\.lastname.example.com.

MAILB lookups would result in either a MR record or one or more MG or MB records. An MR record contained a single mailbox domain name that signified that the queried mailbox domain name was an alias for the one returned in the record. An MB record contained a single hostname of a mail server accepting mail for that mailbox. Multiple MB records was allowed. An MG record contained a single mailbox domain name of a mailbox that should receive mail for the mailbox domain name queried. MG records were supposed to signify that the mailbox domain name was in fact a mailing list. Multiple MG records would be returned and the sender would then send a copy of the mail to each of the listed recipients.
